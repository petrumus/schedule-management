# Feature 07: Invite Link Management

## Overview

Admins and managers can generate, view, and revoke invite links. Each link is a URL with a UUID token that grants new users access to the platform upon sign-up.

## User Stories

### US-7.1: Generate Invite Link
**As** an admin or manager,
**I want** to create an invite link with an expiry date,
**So that** I can onboard new team members.

**Acceptance Criteria:**
- An "Invite Users" button in the header (visible to admin and manager roles) opens the invite management section.
- The generation form has:
  - **Expiry date picker** — required.
  - **Label / note** — optional free-text (e.g. "March 2025 onboarding batch").
  - A "Generate link" button.
- On generate: insert a new row into `invite_links` with `is_active = true`, `use_count = 0`.
- Display the full shareable URL: `https://yourapp.github.io/invite?token=<uuid>`.
- A one-click copy button copies the URL to clipboard.

### US-7.2: View Invite Links
**As** an admin or manager,
**I want** to see all invite links I've created,
**So that** I can track their usage and status.

**Acceptance Criteria:**
- A table shows all links created by the current user.
- Admins see all links (regardless of creator).
- Columns: label, created at, expires at, use count, status (active/expired/revoked).
- Status is derived: "revoked" if `is_active = false`, "expired" if `expires_at < now()`, otherwise "active".

### US-7.3: Revoke Invite Link
**As** an admin or manager,
**I want** to revoke an invite link,
**So that** it can no longer be used.

**Acceptance Criteria:**
- Each active link row has a "Revoke" button.
- Clicking revoke sets `invite_links.is_active = false`.
- The link immediately shows as "Revoked" in the table.
- Only the creator or an admin can revoke a link.

## Database Dependencies

- `invite_links` — full CRUD for admins/managers

## RLS Policies

- Admins and managers can insert and read all invite links.
- Anyone (including unauthenticated) can read a single row by `id` (for token validation during sign-up).
- Only the creator or an admin can update `is_active`.

## Technical Notes

- The invite management can be a modal/dialog opened from the header or a dedicated section within the admin page.
- The UUID for the invite token is the `id` column (PK), generated by Supabase/PostgreSQL.
- Copy-to-clipboard uses the Clipboard API (`navigator.clipboard.writeText()`).
- Date picker for expiry should default to 7 days from now as a reasonable starting point.
